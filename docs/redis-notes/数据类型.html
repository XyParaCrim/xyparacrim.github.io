# Redis数据结构

👋👋 Redis一共有5种数据结构分别为:

  • 字符串（String）
  • 列表（List）
  • 字典（Hash）
  • 集合（Set）
  • 有序集合（Zset）

下面将从书中提取一些要点或记录一些容易遗漏的要点

# String - 字符串

Redis的字符串为动态字符串，是可以修改字符串，内部结构的实现类似于Java的ArrayList：采用预分配冗余空间的方式来减少内存的频繁分配。

当字符串小于1MB时，扩容为加倍现有空间；当字符串大于1MB时，一次只会最多扩容1MB

字符串最大长度为512MB

# Cli

可以使用下面命令操纵：

  • 单键操纵
    • set ${key} ${value}：设置key=value
    • get ${key}：获取key的值
    • del ${key}：删除key的值
  • 批量操作
    • mset ${key1} ${value1} ${key2} ${value2}：批量设置key1、key2为value1、value2
    • mget ${key1} ${key2}：批量删除key1、key2
  • 期限操作
    • expire ${key}：设置key 5s后过期
    • setex ${key} ${expire} ${value}：设置key=value，expire秒后过期
  • 计数操作
    • incr ${key}：对值为整数类型的key，自增一

自增的范围为signed long，即-2^63 ~ 2^63 - 1

String由多个byte（8bit）组成，所以是很多bit组成，即位图（bitmap）

# 常见用途

  • 缓存用户信息，例如将信息以JSON的形式序列化存至Redis中
  • 缓存Java Bean，Spring-web-session默认缓存Java bean作为web session

# List - 列表

Redis与Java里的LinkedList相同，所以它不是RandomAccess类型，支持双向遍历。

List宏观与LinkedList，但是其链表的每个节点其实是压缩列表（ziplist）：

  • 一块连续内存
  • 所有元素彼此紧挨着一起存储

为什么要使用ziplist呢？

  • 节点需要附加指针空间，加重内存碎片化
  • 节省空间

# Cli

  • 基本操作
    • llen ${key}：获取key队列数量
  • 先进先出
    • rpush ${key} ${value1} ${value2}：将value1、value2入key的队
    • lpop ${key}：推出最后一个元素
  • 后进先出
    • rpop：推出队头元素
  • 队列访问 慢操作
    • lindex ${key} ${index}：获取key队列中index位置的元素
    • lrange ${key} ${index-start} ${end-start}：获取key队列中index-start至index-end范围的元素
    • ltrim ${key} ${index-start} ${end-start}：修剪key队列中index-start至index-end范围的元素

expire针对整个List，Hash、Set和Zset也一样

# 常见用途

  • 异步队列使用，将需要延后处理的任务结构序列化成字符串，存储至Redis

# Hash - 字典

Hash与Java中HashMap相似：

  • 无序
  • 数据 + 链表

与HashMap不同的是rehash的过程，java是一次性rehash，而Redis为了追求高性能，不阻塞服务，采用渐进式rehash策略，这与Java 11中ConcurrentHashMap采取的策略是一样的。

  • Redis单线程：定时任务或者hash请求操作中，循序渐进地将旧hash slot搬移至新hash中，将任务分散至每次定时任务和请求操作中。
  • Java11：多线程会在访问Hash时，分到均匀的Slot数量（步长）进行rehash

# Cli

  • hset ${key} ${slotKey} ${slotValue}：设置key字典，slotKey = slotValue
  • hgetall ${key}：获取key的所有元素
  • hlen ${key}：获取key的数量
  • hget ${key} ${slotKey}：获取key中slotKey的值
  • hmget和hmset：批量设置或获取
  • hincrby ${key} ${slotKey} ${count}：自增

# Set - 集合

Set与Java中HashSet相似

  • 无序

# Cli

  • sadd ${key} ${value} ${value1}：添加value、value1至key中
  • smembers ${key}：获取所有元素
  • susmember ${key} ${value}：查询value元素
  • scard ${key}：获取长度
  • spop ${key}：弹出元素

# 常见用途

  • 储存中奖用户ID，去重保证不会用户两次中奖

# Zset - 有序集合

Set类似于Java中HashSet和SortedSet的结合体，它为每个value赋予一个score，用于其的有序，很容易想到其实现的数据结构是跳跃列表

首先元素位于L0层概率为100%，而L1层概率为50%，依次筛选和类推，L2概率25%，L3概率12.5%，所以很多元素可以通过几层就可以访问到。

# Cli

  • zadd ${key} ${score} ${value}：设置元素和权重
  • zrange ${key} ${index-start} ${index-end}：按照权重获取范围排序元素
  • zrevrange ${key} ${index-start} ${index-end}：按照权重获取范围逆序排序元素
  • zscore ${key} ${value}：获取权重值
  • zrank ${key} ${value}：获取value的排名
  • zrangeByscore ${key} ${score-start} ${score-end}：获取权重范围的元素
    • zrangeByscore ${key} ${score-start} ${score-end} withscores：withscores表示同时返回参数
  • zrem ${key} ${value}：删除value

inf表示无穷，-inf表示负无穷

# 常见用途

  • 储存粉丝列表，score为关注时间，使用关注时间进行排序
  • 储存学生成绩，score为考试成绩
