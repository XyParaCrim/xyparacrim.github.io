# Redis内存对象

Redis是如何在内存中储存大量数据的？

# 位数

  • 内存不超过4G，考虑是32bit进行编译
    • 可使用的指针空间少一半

# ZipList

暂缺

# Key回收

Redis将设置过期时间的Key放入一个独立的字典里，会通过定时加惰性删除的方式，清理Key

# 惰性删除

客户端访问Key时，发现Key过期，则将其删除

# 定时删除

Redis默认每秒进行10次过期扫描，但是不会全遍历字典，而是采用贪心策略：

  1. 从过期字典中，随机选出20个Key
  2. 删除这20中过期的Key
  3. 若过期比例超出1/4，重复1

每次扫描限制在25ms中

从节点不会主动扫描，会根据主节点的del指令

# 异步回收

  • unlink指令可以切断key和value的联系，执行异步回收
  • AOF可以异步执行
  • flushall async

配置

  1. slave-lazy-flush：从节点接收完rdb文件后的flush操作
  2. lazyfree-lazy-eviction：内存达到maxmemory时进行淘汰
  3. lazyfree-lazy-expire key：过期删除
  4. lazyfree-lazy-server-del rename：指令删除
