Redis延时队列

XyParaCrim 2020-12-13 Redis

# 延时队列

😐😐

RabbitMQ和kafka作为传统消息中间件，支持的特性非常多，但是Redis的队列可以作为轻量级消息队列

# 简单队列

  • rpush：队尾
  • rpop：队尾
  • lpush：队头
  • lpop：队头

当队列空

  • 可以让客户端Tread.slepp(1000)的线程沉睡，减少客户端CPU消耗，减轻Redis的QPS
  • 使用blpop/brpop进行阻塞读队列消息

::: error 使用blpop/brpop时，若连接闲置过长，服务器将主动断开，客户端抛出异常，需要添加处理手段 :::

# 代码实现

  • zset作为数据结构
  • score存放入队时间
import com.alibaba.fastjson.JSON;
import com.alibaba.fastjson.JSONObject;
import lombok.RequiredArgsConstructor;
import redis.clients.jedis.Jedis;

import java.util.Set;
import java.util.UUID;
import java.util.concurrent.locks.LockSupport;
import java.util.function.Consumer;

@RequiredArgsConstructor
public class RedisDelayQueue<T> implements Runnable {

    @RequiredArgsConstructor
    private static class Node<T> {

        private final String id = UUID.randomUUID().toString();

        private final T value;

    }


    private final Jedis jedis;

    private final Consumer<T> handler;

    private final String key = UUID.randomUUID().toString();


    public void add(T value) {
        jedis.zadd(
                key,
                System.currentTimeMillis() + 5000,
                JSON.toJSONString(new Node<>(value))
        );
    }

    @Override
    public void run() {
        while (!Thread.currentThread().isInterrupted()) {
            Set<String> values = jedis.zrangeByScore(key, 0, System.currentTimeMillis(), 0, 1);

            if (values.isEmpty()) {
                LockSupport.parkUntil(500);
                continue;
            }

            String s = values.stream().findFirst().get();
            if (jedis.zrem(key, s) > 0) {
                @SuppressWarnings("unchecked")
                Node<T> node = (Node<T>) JSONObject.parseObject(s, Node.class);

                handler.accept(node.value);
            }
        }
    }


}
