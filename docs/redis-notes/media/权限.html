# 权限

# 背景

解释权限管理的名词

# 权限（Permission）

由操作+资源构成一个权限，简单地定义一条权限：

@JalorResource(code = "People", desc = "People Manager")
public class PeopleService {

    @JalorOperation(code = "read", desc = "Read Records")
    public int peopleCount() {
        return 0;
    }

}

@JalorResource定义了People资源，@JalorOpertaion定义Read操作，构成一条权限

公共服务会在启动时收集@JalorResource注解的类，再寻找该类下的@JalorOpertaion注解的方法进行注册和持久化。而其他微服务，需要在管理界面->角色管理->授予权限->同步权限，才能通过调用已注册到公共服务的其他微服务的请求注册，或者调用公共服务的OperationService的相关方法同步权限

# 角色（Role）

角色主要由零个或多个不同的权限（Permission）组成，例如角色A具有对People资源的Read、Write和Update三个操作权限。

在公共服务的管理界面添加角色，并为角色赋予权限（Permission）集合

# 数据维度（Dimension）

数据纬度其实就是一个数组，用于SQL语句的in关键字：

select id, people_name, people_age from people_table where people_age in (33, 34, 35)
1

(33, 34, 35)就是一个数据维度。

在公共服务的管理界面配置数据维度，配置需要一个唯一的名称（code）+ 请求（url，返回数组的请求）

# 数据范围（Program）

数据范围由零个或多个数据维度（Dimension）组成，在公共服务管理界面配置，配置需要一个唯一的名称（code）和多个数据维度组成

数据范围用于筛选行数据。假设数据范围old-people为(33, 34, 35)，MyBatis的mapper中：

<select>
   select id, 
          people_name, 
          people_age 
   from people_table 
   where <add-program  old-people:people_age/>
</select>

# 用户（User）和群组（Group）

用户和群组在Jalor的用法完全一致，群组只是多个用户

每个用户和群组都可以添加零个或多个不同的权限点，权限点由一个角色 + 数据范围组成。

在公共服务管理界面定义用户和群组名，再添加权限点。

按照上面的例子，用户由权限点组成，即角色跟数据范围，角色约束了资源种类（People）和操作（Read），数据范围筛选了资源（33 ～ 35岁）。

# 如何实现

在Jalor中沿用SpringMVC模式，处理一个请求在一个线程中完成，所以在运行时处理请求持有一个RequestContext请求上下文，里面包含了：

  • 用户信息
  • 用户当前角色
  • 用户担任哪些有效角色
  • 角色对应的数据范围。

沿用上面的例子：

@JalorResource(code = "People", desc = "People Manager")
public class PeopleService {

   private final PeopleRepository repository;

    @JalorOperation(code = "read", desc = "Read Records")
    public int peopleCount() {
        return repository.findCount();
    }

}

角色用法： 当进入方法之前会首先进入一个AOP拦截器方法，主要根据配置的操作码和资源码等组成唯一标示，在RequestContext中匹配用户担任的当前角色是否有权限（Permission）的，若匹配则说明有权限，否则抛出异常返回无权限。

数据范围用法： MyBatis的mapper里使用数据范围，需要声明使用的数据范围和筛选的字段。使用的数据范围，运行时会根据RequestContext请求上下文中当前角色和角色包含数据范围，是否一致，决定是否符合权限执行。
